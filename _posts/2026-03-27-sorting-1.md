시간 복잡도가 O($n^2$)인 대표적인 정렬 알고리즘에 대해 알아보자.

<br>
## 1. 순차 정렬 / Sequential sort

가장 기본적인 정렬 알고리즘으로, 맨 앞에서부터 제일 작은 원소를 배치하게 만들어 나가는 알고리즘이다.

아래는 C로 구현한 순차 정렬이다.

```c
#define swap(a, b) {int temp = a; a = b; b = temp;} 

void sequential_sort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        for (int j = i+1; j < n; j++) {
            if (arr[i] > arr[j]) {
                swap(arr[i], arr[j]);
            }
        }

    }
}
```

특징으로는,

1. 시간 복잡도는 최선/최악의 경우 모두 $O(n^2)$이다.
2. 동일한 값을 가지는 원소 간의 순서를 보존하지 않는 **불안정 정렬(Unstable sort)**이다. 
    
    ([3, 2, 2, 1]를 정렬하는 경우를 생각해보자.)
    
3. 외부 메모리를 사용하지 않는다. (제자리 정렬 / In-place sorting 이다.)
4. 두 원소를 교환(swap)하는 연산이 매우 빈번하게 일어나 효율성이 떨어지기 떄문에 잘 사용하지 않는다.

<br>
## 2.  버블 정렬 / Bubble sort

이웃한 두 원소를 비교하며 정렬하는 알고리즘으로, 한 번의 cycle이 지나면 제일 큰 값이 맨 뒤에 위치하게 된다.

아래는 C로 구현한 버블 정렬이다.

```c
#define swap(a, b) {int temp = a; a = b; b = temp;} 

void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) { 
        for (int j = 0; j < n-i-1; j++) {
            if (arr[j] > arr[j+1]) {
                swap(a[j], a[j+1]);
                is_swapped = 1;
            }
        }
    }
} 
```

특징으로는,

1. 시간 복잡도는 최선/최악의 경우 모두 $O(n^2)$이다.
2. 동일한 값을 가지는 원소 간의 순서를 보존하는 **안정 정렬(Stable sort)**이다.
3. 외부 메모리를 사용하지 않는다.
4. 한번의 cycle에서 원소의 교환이 이루어졌는지 확인하는 flag 변수를 두면, 시간 복잡도가 $O(n)$으로 개선된다.

아래는 개선된 버블 정렬이다.

```c
#define swap(a, b) {int temp = a; a = b; b = temp;} 

void bubble_sort_improved(int arr[], int n) {
    for (int i = 0; i < n-1; i++) { 
        int is_swapped = 0; // for early exit
        for (int j = 0; j < n-i-1; j++) {
            if (arr[j] > arr[j+1]) {
                swap(arr[j], arr[j+1]);
                is_swapped = 1;
            }
        }
        if (!is_swapped) {
            return;
        }
    }
}
```

이미 정렬된 데이터에 대해, 개선된 버블 정렬은 이웃한 원소들끼리의 대소 비교를 한 번 진행하고 반복문을 탈출한다.

<br>
## 3.  선택 정렬 / Selection sort

순차 정렬과 동일하게 맨 앞에서부터 제일 작은 원소를 배치해 나가는 알고리즘이지만, 한 번의 cycle에 swap이 1번만 발생한다는 점에서 순차정렬과 다르다.

 아래는 C로 구현한 선택 정렬이다.

```c
#define swap(a, b) {int temp = a; a = b; b = temp;}

void selection_sort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        int min_idx = i;
        for (int j = i+1; j < n; j++) {
            if (arr[min_idx] > arr[j]) {
                min_idx = j;
            }
        }
        swap(arr[min_idx], arr[i]);
    }
}
```

특징으로는,

1. 시간 복잡도는 최선/최악의 경우 모두 $O(n^2)$이다.
2. 불안정 정렬이다.
3. 외부 메모리를 사용하지 않는다.

<br>
## 4. 삽입 정렬 / Insertion sort

삽입 정렬은 점진적으로 정렬 범위를 넓혀 나가는 알고리즘으로, 정렬을 해야하는 새로운 원소가 정렬되어 있는 부분을 탐색하며 자신보다 작은 원소를 찾을 때까지 이동하며 자리를 교환한다.

```c

void insertion_sort(int arr[], int n) {
    if (n < 2) return;

    int key, j;
    for (int i = 1; i < n; i++) {
        key = arr[i];
        for (j = i; j > 0 && arr[j-1] > key; j--) {
            arr[j] = arr[j-1];
        }
        arr[j] = key;
    }
}
```

특징으로는,

1. 시간 복잡도는 최선의 경우 $O(n)$, 최악의 경우 $O(n^2)$ 이다.
2. 안정 정렬이다.
3. 외부 메모리를 사용하지 않는다.
4. 삽입 정렬은 상당 부분 정렬되어 있는 배열에 대해서는 $O(nlogn)$의 시간 복잡도를 가지는 정렬 알고리즘(퀵 정렬, 힙 정렬, 합병 정렬)보다 더 빠르게 작동할 수 있다.
    
    (실제로 Python의 정렬 알고리즘(Timsort) 이나 C++ STL의 std::sort(Introsort) 는 두 방식을 섞어서 사용한다.)
