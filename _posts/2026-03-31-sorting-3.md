시간 복잡도가 $O(n^2)$, $O(nlogn)$가 아닌 기타 정렬 알고리즘들에 대해 알아보자.

<br>
## 1. 쉘 정렬 / Shell sort

쉘 정렬은 삽입 정렬의 성질을 이용하는 일반화된 삽입 정렬로, 일정 간격(gap) 떨어져 있는 두 원소를 비교하면서 정렬하는 알고리즘이다. Gap은 점진적으로 줄어들며, 마지막에는 1이 되어 삽입정렬과 동일하게 작동한다.

쉘 정렬의 아이디어는 다음과 같다.

- 삽입 정렬은 **거의 정렬된 데이터**에 대해 굉장히 빠르다.
    
    쉘 정렬은 이 삽입 정렬의 성질을 이용한다. 
    
    Gap 만큼 떨어져 있는 원소들을 그룹으로 만들고, 그 그룹에 대해 삽입 정렬을 수행한다. 이 과정에서 큰 값들은 뒤로, 작은 값들은 앞쪽으로 이동하게 된다. 그 결과, 마지막 단계인 gap = 1에 도달했을 때는 이미 **거의 정렬된** 상태의 데이터가 되어 있어 매우 적은 비교만으로 정렬이 끝나게 된다.
    

아래는 C로 구현한 쉘 정렬이다.

```c
void shell_sort(int arr[], int n) {
    for (int gap = n/2; gap > 0; gap /= 2) {
        for (int i = gap; i < n; i++) {
            int key = arr[i];
            int j;
            for (j = i; j >= gap && arr[j - gap] > key; j -= gap) {
                arr[j] = arr[j - gap];
            }
            arr[j] = key;
        }
    }
}
```

특징으로는, 

- 성능이 **어떠한 수열을 Gap Sequence로 쓰냐에 따라 달라진다.**
    
    여기서 gap sequence는 gap으로 사용할 값들을 모아 놓은 수열이다.
    
    - Shell의 방식 : $n/2$ → $n/4$ → … → $1$ , Worst-case time complexity는 $O(n^2)$이다.
    - Knuth의 방식 : $h = 3h + 1$, Worst-case time complexity는 $O(n^{3/2})$이다.
    - Ciura의 방식 : 실험적으로 최적화된 수치 (1, 4, 10, 23, …)을 사용
    
    다른 gap sequence들도 있으니, 궁금하다면 아래를 참고하자.
    
    https://en.wikipedia.org/wiki/Shellsort#Gap_sequences
    
- **불안정 정렬**이다.
- **In-place sorting(제자리 정렬)**이다. 즉, 추가적인 공간을 필요로 하지 않는다.

## 2. 계수 정렬 / Counting sort

계수 정렬의 시각화 : https://www.cs.usfca.edu/~galles/visualization/CountingSort.html

계수 정렬은 영문명에서 알 수 있듯이, 정렬하고자 하는 배열 내부의 key들의 개수를 세고, 누적합(prefix-sum)을 통해 **비교 없이** 정렬하는 알고리즘이다. 배열의 인덱스를 사용하는 알고리즘의 특징 상, 실수 배열은 정렬하지 못하고 **오직 정수 배열만 정렬 가능**하다.

알고리즘의 핵심 아이디어는 다음과 같다.

1. 비교가 아닌 개수를 세자.
    
    대부분의 정렬 알고리즘은 A가 B보다 큰지 빈번하게 비교하지만, 계수 정렬은 비교를 하지 않고 데이터의 값이 무엇인지 확인하고, 그 값에 해당되는 인덱스의 값을 +1 함으로서 개수를 센다.
    
    예를 들자면, 배열에 3이라는 값이 있다면, count라는 개수를 담는 배열이 존재한다고 했을 때, count[3]의 값을 +1 한다.
    
2. 누적합 (Prefix sum)
    
    개수를 알고 있으면 사실상 정렬은 가능하다. 개수만 세어 앞에서부터 채워 넣으면 되기 때문이다. 하지만, 불안정 정렬이 될 수 있다. 
    
    이를 해결하기 위해, 누적합 배열을 만들고 배열을 역순으로 순회하며 기존 배열의 특정 값이 결과 배열에 위치할 **절대적인 위치값**을 계산한다.
    

계수 정렬은 크게 4단계의 과정을 거쳐 완성된다. 이해를 돕기 위해 배열 `A = [2, 5, 3, 0, 2, 3, 0, 3]`을 정렬한다고 가정한다.

### **Step 1: 빈도수 계산 (Counting)**

원본 배열 `A`를 처음부터 끝까지 순회하며, 각 원소의 값을 인덱스로 사용하여 `Count` 배열의 값을 증가시킨다.

- `A`에서 `0`이 2개, `2`가 2개, `3`이 3개, `5`가 1개 발견됨.
- **Count 배열:** `[2, 0, 2, 3, 0, 1]` (인덱스 0부터 5까지)

### **Step 2: 누적합 계산 (Prefix Sum)**

`Count` 배열의 각 요소를 누적해서 더하여 누적합을 계산한다. 주목할 점은 누적합 배열의 각 값들은 해당 숫자가 정렬된 배열에서 들어갈 **마지막 위치**를 의미한다.

- `Count[i] = Count[i] + Count[i-1]` 수행
- **누적 Count 배열:** `[2, 2, 4, 7, 7, 8]`
    - *해석: 숫자 '3'까지의 누적합이 7이므로, 3개의 ‘3’ 중 마지막 '3'은 결과 배열의  7번 인덱스에 위치해야 함.*

### **Step 3: 역순 순회 및 배치 (Placement)**

원본 배열 `A`의 **뒤에서부터(오른쪽에서 왼쪽으로)** 원소를 하나씩 확인하며 결과 배열 `B`에 배치한다.

1. 원본 배열의 값 `A[i]`를 확인한다.
2. `Count[A[i]]`의 값을 1 감소시킨 후, 그 값을 인덱스로 사용하여 `B`에 저장한다.
    - *이때 1을 먼저 감소시키는 이유는 배열의 인덱스가 0부터 시작하기 때문이다.*

### **Step 4: 복사 (Copy)**

정렬이 완료된 임시 배열 `B`의 내용을 원래의 배열 `A`로 복사하여 정렬을 마무리한다.

아래는 C로 구현한 계수 정렬이다.

```c
#include <stdio.h>
#include <stdlib.h> // for malloc & calloc
#include <string.h> // for memcpy

void counting_sort(int* arr, size_t n) {
    if (n <= 1) return;

    int max = arr[0];
    for (size_t i = 1; i < n; i++) {
        max = max > arr[i] ? max : arr[i];
    }

    int* count = (int*) calloc(max + 1, sizeof(int));
    int* temp = (int*) malloc(n * sizeof(int));
    if (count == NULL || temp == NULL) {
        fprintf(stderr, "Memory allocation failed!\n");
        free(count);
        free(temp);
        return;
    }

    for (size_t i = 0; i < n; i++) ++count[arr[i]]; // count keys

    for (int i = 1; i < max + 1; i++) count[i] += count[i-1]; // make prefix-sum array
    // assume we only have non-negative key values

    for (int i = n-1; i >= 0; i--) temp[--count[arr[i]]] = arr[i];

    memcpy(arr, temp, n * sizeof(int));
    free(temp);
    free(count);
}
```

계수 정렬은 count 배열의 인덱스를 사용하므로 기본적으로 음수 정렬이 불가능하다. 하지만 **배열의 최솟값(min)을 0으로 만드는 오프셋(offset)**을 적용하면 음수 범위까지 간단히 확장할 수 있다.

아래는 범위를 음수까지 확장한 계수 정렬이다.

```c
void counting_sort_generalized(int* arr, size_t n) {
    if (n <= 1) return;

    int min = arr[0];
    int max = arr[0];
    for (size_t i = 1; i < n; i++) {
        max = max > arr[i] ? max : arr[i];
        min = min < arr[i] ? min : arr[i];
    }

    size_t range = max - min + 1;
    
    int* count = (int*) calloc(range, sizeof(int));
    int* temp = (int*) malloc(n * sizeof(int));
    if (count == NULL || temp == NULL) {
        fprintf(stderr, "Memory allocation failed!\n");
        free(count);
        free(temp);
        return;
    }

    for (size_t i = 0; i < n; i++) ++count[arr[i] - min]; // count keys

    for (int i = 1; i < range; i++) count[i] += count[i-1]; // make prefix-sum array

    for (int i = n-1; i >= 0; i--) temp[--count[arr[i] - min]] = arr[i];

    memcpy(arr, temp, n * sizeof(int));
    free(temp);
    free(count);
}

int main() {
    int arr[] = {4,-8,2,2,0,-4,-9,33,1,-2};
    int n = sizeof(arr)/sizeof(arr[0]);
    counting_sort_generalized(arr, n);
    
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
}

// 결과 : -9 -8 -4 -2 0 1 2 2 4 33
```

특징으로는,

- 시간 복잡도와 공간 복잡도 모두 **$O(n + k)$**이다. (n : 데이터의 개수, k는 데이터의 범위)
- **데이터 범위(k)가 데이터 개수(n)보다 과도하게 크면 메모리 낭비가 매우 심해진다**. 따라서 데이터의 밀도가 높은 상황에서만 사용하는 것이 적절하다.
- 역순 순회 방식을 사용할 경우 동일한 키 값을 가진 원소들의 순서가 유지되는 **안정 정렬**이다.

## 3. 기수 정렬 / Radix sort

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

/*

0 | 10, 40, 30, 120, ...
1 | 31, 21, ...
2 | 92, 2, ... 
... 이런 형태의 버킷을 1차원 배열로 만들었다고 생각하면 됨

[10, 40, 30, 120, ... , 31, 21, ... , 92, 2, ... , ... ] 이렇게
[       자리수 0      ] [  자리수 1  ] [   자리수 2   ] ...
       
*/

void radix_sort(int* arr, int n, int base) {
    int* output = (int*) malloc(n * sizeof(int));
    int* count = (int*) malloc(base * sizeof(int));
    
    // find max value
    int max = arr[0];
    for (int i = 1; i < n; i++) max = max > arr[i] ? max : arr[i];

    for (long long pow = 1; max >= pow; pow *= base) {
        memset(output, 0, n * sizeof(int));
        memset(count, 0, base * sizeof(int));

        // count frequency of current digit
        for (int i = 0; i < n; i++) ++count[(arr[i] / pow) % base]; 

        // make prefix-sum array
        for (int i = 1; i < base; i++) count[i] += count[i-1];

        // 뒤에서부터 순회하며 output 배열에 배치 (계수 정렬과 동일)
        for (int i = n-1; i >= 0; i--) {
            output[count[(arr[i] / pow) % base] - 1] = arr[i];
            --count[(arr[i] / pow) % base];
        }

        memcpy(arr, output, n * sizeof(int));
    }

    free(output);
    free(count);
}

int main() {
    int arr[] = {10, 23, 231, 13, 41, 539, 120, 155, 153, 2984};
    int n = sizeof(arr)/sizeof(arr[0]);
    radix_sort(arr, n, 10);

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
}
```
